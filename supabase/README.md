# Local Supabase Database

This project uses [Supabase CLI](https://supabase.com/docs/guides/cli) to manage the local development environment.

## Quick start commands

### 1. Start the environment

To spin up the Docker containers and the local Supabase API:

```bash
supabase start
```

### 2. Reset the database (Schema + Seed)

If you have modified the schema (migrations) or the initial data, apply the changes with:

```bash
supabase db reset
```

*This command wipes the local database and re-populates it from scratch based on your migration files and the `seed.sql` file.*

> **Note on Seeding:** The `seed.sql` file is automatically generated by aggregating the contents of the `/seeds` folder. Ensure your data scripts are correctly placed there before running a reset.

### 3. View local credentials

To configure your `.env` file, run the following command to retrieve your local project details:

```bash
supabase status
```

**Required Environment Variables:**
Configure your `.env` file with these values:

* `NEXT_PUBLIC_SUPABASE_URL=http://127.0.0.1:54321`
* `NEXT_PUBLIC_SUPABASE_ANON_KEY=[YOUR_S3_SECRET_KEY]` (Obtained via the initial `supabase start` setup)

### 4. Stop the Environment

To stop the containers without deleting your local data:

```bash
supabase stop
```

---

## File structure

* `/migrations`: Defines the database structure (tables, relationships, constraints).
* `/seeds`: Contains individual SQL files that are aggregated into `seed.sql`.
* `seed.sql`: The primary seed file used to populate the database. It is auto-generated from the contents of the `/seeds` folder to ensure the project is functional immediately upon setup.
